## 문제 상황

클로드 코드를 실행했는데 갑자기 아래와 같은 에러가 발생했다.

```
⎿ API Error: 404
{ "type":"error",
"error":{
	"type":"not_found_error",
	"message":"model: claude-sonnet-4-20250514"
},
"request_id":"..."
}
```

### 원인

내가 사용하던 `claude-sonnet-4-20250514` 모델이 6월 15일자로 deprecated되어 더 이상 API에서 사용할 수 없었다. <br/>
[문서](https://platform.claude.com/docs/ko/about-claude/model-deprecations)에서는 `claude-sonnet-4-6`으로 마이그레이션을 권장하고 있었다.

### 처음에 시도한 방법

터미널에서 아래와 같이 실행했다.

```
claude --model claude-sonnet-4-6
```

![Image](/assets/images/26/0616-1.png)

이후 클로드 코드 내부에서 `/model` 로 확인해보니 Sonnet 4.6이 제대로 적용되어 있었지만, 클로드 코드를 종료한 뒤에 다시 실행하면 원래 버전으로 돌아가면서 다시 404 에러가 발생했다. <br/>
`--model` 플래그는 현재 실행한 세션에만 적용되는 일회성 용도라고 한다. ([문서](https://support.claude.com/en/articles/11940350-claude-code-model-configuration))

### 해결 방법

클로드 설정 파일인 `~/.claude/settings.json`을 확인해보니 비어있었다. 아래와 같이 모델 설정을 추가해주고 실행하니 해결되었다.

```
{
	"model": "claude-sonnet-4-6"
}
```

어째서 그동안 `claude-sonnet-4-20250514` 를 계속 default 모델로 사용하고 있었던 걸까 <br/>
--> 클로드 코드 버전이 낮았기 때문인 듯하다... `brew upgrade claude-code` 로 버전 업데이트를 해주고 나니, `settings.json`을 다시 비워두어도 Sonnet 4.6이 default 모델로 잘 잡힌다.
